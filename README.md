# Gym Membership Database
SQL database created with 2 other of my peers, Dorian Mkam, and Tidjani Ahmed. Sample data was collected from Kaggle.

## Overview
This project is a relational database and was designed to see trends and patterns of gym members according to different biometric data. The design follows 3NF and enforced referenital integrity through primary and foreign key constraints. 

## Technologies
- EXCEL
- MYSQL WORKBENCH

## Tables 
- **Members** – Core gym member data and references to demographic attributes
- **Gender** – Gender table
- **WorkoutFrequency** – Workout frequency table
- **ExperienceLevel** – Member experience levels
- **BioMetrics** – Member biometric data (BMI, weight, body fat, etc.)
- **WorkoutType** – Types of workouts performed (HITT, Cardio, Yoga, Stength)
- **WorkoutSession** – Biometric tracking during workouts (calories burned, heartrate.)
